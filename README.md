# 개발 명세서 
1. 실시간 시각 피드백 (Alpha 처리)
로직: votedSide 상태에 따라 선택된 상품과 비선택 상품의 시각적 대비 극대화.

선택 상품: opacity-100, 채도 강조, 테두리 하이라이트 적용.

비선택 상품: opacity-30, blur(4px) 처리, 그레이스케일 적용으로 시각적 차단.

미선택 상태: 양측 모두 opacity-100으로 균형 유지.

2. Swipe 내비게이션 (모바일 최적화)
구현: onTouchStart, onTouchEnd 이벤트를 이용한 거리(deltaX) 계산.

동작:

Left Swipe: nextSet() 호출 (다음 비교 세트로 이동).

Right Swipe: prevSet() 호출 (이전 비교 세트로 이동).

전환: Framer Motion 등을 활용한 부드러운 슬라이딩 애니메이션 권장.

3. 하이브리드 반응형 레이아웃 (자동 전환)
세로 모드 (Portrait): 좌/우 분할 배치 (flex-row). 좁은 폭에서 두 상품을 한눈에 비교.

가로 모드 (Landscape): 상/하 분할 배치 (flex-col). 와이드 화면을 활용해 이미지/영상을 크게 노출.

감지: window.onresize 또는 matchMedia를 사용하여 isPortrait 상태 실시간 업데이트.

4. 실시간 초단위 카운트다운 (T-minus)
데이터: 각 세트별 endTime (타임스탬프) 기반.

구현: setInterval을 활용해 1초마다 잔여 시간 계산 및 UI 렌더링.

포맷: HH : mm : ss (시간 : 분 : 초) 형태의 긴박감 있는 타이머 노출.

마감 로직: 시간이 0이 되면 즉시 투표를 차단하고 'Winner' 엠블럼 자동 노출.

5. 핵심 인터랙션 요약
참여 방식: 화면 Double Tap 시 투표 완료 및 하트 애니메이션 실행.

정보 노출: 화면 중앙에 실시간 참여자 수 및 득표율(% 바) 상시 노출.

제한: 최대 5개 세트 롤링 구조 유지.

Double Tap 시: 짧고 강한 진동(Haptic Feedback)을 주어 참여 완료를 시각 외적으로도 인지시켜야 합니다.

사운드: 2분할 영상의 소리가 섞이지 않도록, 사용자가 마지막으로 터치하거나 시선이 머무는 쪽의 볼륨을 미세하게 키우는 로직을 검토해 보세요.
